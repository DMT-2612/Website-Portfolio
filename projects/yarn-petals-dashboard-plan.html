Yarn Petals Dashboard Plan

As-built Power BI dashboard structure, model design, layout, and validation baseline.

1. Dashboard Purpose

The dashboard was designed as a one-page executive sales report for Ivy / Yarn Petals. Its purpose is to help the owner review sales performance, order volume, customer count, sales channel mix, product mix, and category performance from one structured dataset.

2. Scope

In Scope

  • Load `Universal.xlsx` sheet `Orders` into Power BI.
  • Use a simplified model with Orders, DimDate, Metrics, and Parameter tables.
  • Create three fixed KPI measures: Net Sales, Orders Count, and Total Customers.
  • Use a field parameter to switch charts between Net Sales and Orders Count.
  • Build one report page with 16 visuals and Yarn Petals branding.
  • Keep Email in the model but exclude it from all report visuals.

Out of Scope

  • Prior-year, YoY, MoM, QTD, or MTD time intelligence.
  • Separate dimension tables for product, animal, colour, occasion, channel, or state.
  • Power BI Service / Fabric publishing.
  • Row-level security.
  • Real-time Shopify API integration.

4. Data Model

The Power BI model uses a simple reporting structure:

  • `Orders`: source table from the universal dataset.
  • `DimDate`: date table for slicer and Year-Month trend axis.
  • `Metrics`: central table for DAX measures.
  • `Parameter`: field parameter for Net Sales vs Orders Count chart switching.

`DimDate[Date]` connects to `Orders[Date]` using a single-direction one-to-many relationship. Business rules are applied upstream before the data reaches Power BI.

KPI cards stay fixed. The field parameter only changes chart values, so users can compare sales value and order volume without changing the core KPI section.

The report uses a 1280 x 720 canvas and is structured into clear dashboard zones.

7. Visual Inventory

  • One title text box.
  • One parameter slicer.
  • Six report slicers.
  • One KPI multi-card.
  • One sales channel donut.
  • One monthly trend column chart.
  • One state bar chart.
  • Four distribution donuts for colour, product, animal, and occasion.

8. Definition of Done

  • Unfiltered KPI totals match the validation baseline.
  • Six filters and the parameter slicer work as expected.
  • Parameter changes donut, channel, state, and trend charts only.
  • Trend uses Year-Month only.
  • KPI cards do not show prior-year or YoY reference labels.
  • Distribution donuts are configured as percent of grand total.
  • Email is not visible on any report visual.
  • Yarn Petals theme is applied consistently.
