Spotted: Phishing Recognition and Response Prototype

A mobile prototype that helps young job seekers practise scam recognition before they respond.

Role: Business analysis & UX design Context: ICT20025 User Experience Design Tool: Figma prototype

Project overview

Spotted is an interactive mobile prototype designed to help students and recent graduates practise phishing recognition and response in job-search scenarios. The prototype presents realistic scam messages, asks users to make a decision, and then provides feedback based on the response path they choose.

Spot scams before they spot you.

Context & problem

Phishing attacks often rely on human decision-making under pressure or uncertainty. In a job-search context, suspicious messages can look convincing because they may use authority, urgency, or attractive employment opportunities.

Students and recent graduates can be especially exposed because they often receive messages from unfamiliar recruiters, job platforms, or organisations. The problem was to design a low-pressure learning experience that helps users practise safer decisions before sharing personal information, clicking links, or responding to suspicious requests.

Target users

  • Students seeking internships, part-time work, or graduate opportunities.
  • Recent graduates applying for entry-level roles.
  • Young job seekers receiving unfamiliar recruiter messages, job offers, or application requests.

Scope

The prototype focuses on a mobile, module-based learning flow. Users move through onboarding, choose a phishing learning module, read a suspicious message without hints, make a response decision, receive outcome-based feedback, and review the evidence behind the result.

The project did not include a live application, backend, real accounts, automated phishing detection, reporting integrations, or production deployment. It was designed in Figma to demonstrate the user experience and decision logic.

Business analysis contribution

  • Framed the product problem around job-search phishing risk and user decision-making.
  • Defined functional and non-functional requirements for the prototype experience.
  • Developed user stories and acceptance criteria for onboarding, module selection, decisions, feedback, and learning review.
  • Mapped the activity flow across scenario types, user responses, scoring outcomes, and the learning loop.
  • Translated learning requirements into a structured Figma prototype with realistic message surfaces and feedback screens.

Key requirements

  • Users must be able to start through a short onboarding flow.
  • Users must be able to view job-search phishing learning modules and scenario status.
  • Users must read the message before seeing hints or red flag explanations.
  • Users must choose how to respond and receive immediate feedback linked to their decision.
  • Users must be able to review red flags, explanations, points, and progress after a scenario.

Outcome

The project resulted in a high-fidelity Figma prototype that demonstrates the core interaction model for phishing recognition and response training. It shows how students and recent graduates can move through a structured learning flow: selecting a module, reading a message without hints, choosing a response, receiving feedback, reviewing the evidence, and returning to module progress.

From a Business Analyst perspective, the project produced clear requirements, user stories with acceptance criteria, and an activity diagram explaining the decision and scoring logic behind the prototype.
